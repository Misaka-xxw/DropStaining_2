# **真实设备接口参数清单（P0-03 现场取证清单）**

> 本文件是 P0-03 的现场取证清单，服务于后续现场核对、工程配置和安全审批。
> 本文件**不**记录任何真实现场敏感参数（真实 COM/IP/NodeID/CAN 参数/坐标/泵映射/安全 IO 极性等）。
> 未现场确认的内容一律标注「待确认」，不得用 SDK 默认值、Demo 配置、历史样例或推测值冒充现场事实。
> P1-01（SDK 运行时部署验证）通过**不等于**真实设备已连接、可通讯、可只读或可运动；P1-02 仍须等待 P0-03 / P0-04 / P0-05 门禁。

### 1.4.1 DCR55 安装与样本扫码职责（当前确认口径）

- DCR55 安装在机械臂末端，由机械臂带动到样本 / 玻片扫码位置；
- 现有坐标表中的 `ArmCamera` 若实际指该随动扫码组件，应按“机械臂安装的样本二维码扫码头”理解，不再解释为独立视觉定位相机；
- 后续命名建议统一为 `DCR55Scanner` 或 `SampleScanner`，避免 `ArmCamera` 被误解为独立相机；
- `ArmCamera` / `DCR55Scanner` 是工具设备或工具参考，不应作为机械臂运动目标点；
- 样本扫码流程应为：机械臂到达对应 Slot / `ScannerRegion` 的扫码目标位，确认运动完成并保持静止，再触发 DCR55 单次扫码；
- 必须标定 DCR55 扫码中心相对于机械臂坐标基准或 Needle1 基准的 X / Y / Z 工具偏移；
- `SampleScan` 不应默认表示每张玻片都经过的固定扫码工位；如确实存在统一校准 / 测试 / 参考点，建议改名为 `SampleScannerCalibrationPoint`。

### 1.5.1 试剂扫码职责（当前确认口径）

- 试剂扫码不使用机械臂末端的 DCR55；
- 试剂扫码由主控板内置多通道二维码扫描模块负责，协议 Parent Command 为 `0x08`；
- 试剂扫码模块固定安装，不随机械臂运动；
- 扫码对象为试剂架中的试剂瓶二维码；
- 协议支持 `ch1–ch16`，当前倾向实际使用 `ch1–ch5` 对应 5 列试剂架，但 `ch1–ch5` 与具体物理列、R 位编号之间的映射仍需现场确认，不能写死；
- 试剂架业务模型当前仍以 `R1–R40` 管理；现场列式位置标记倾向 `R11–R18`、`R21–R28`、`R31–R38`、`R41–R48`、`R51–R58`，需与既有 `R1–R40` 数据模型统一确认；
- 推荐触发时机为试剂架装载后人工触发、运行启动前自动校验、工程或操作员手动重新扫描；不建议运行中无条件反复扫描；
- 扫码结果应通过“扫码通道 + 扫码位置标记”唯一映射到 R 位，再绑定 `ReagentSlot -> ReagentBottle -> ReagentCode -> ReagentType -> RemainingVolume -> Validity`；
- 扫码结果状态建议为 `VALID`、`EMPTY`、`INVALID`；其中 `EMPTY` 和 `INVALID` 的硬件原始返回格式仍需现场确认；
- 启动前必须校验所需试剂的位置存在、二维码有效、试剂类型匹配、未过期、余量足够；缺失、无效或无法查表时必须阻止启动。

| 扫码对象 | 扫码设备 | 是否随机械臂运动 |
|:----|:----|:----:|
| 样本 / 玻片二维码 | DCR55 / `SampleScanner` | 是 |
| 试剂瓶二维码 | 主控多通道扫码模块 `0x08` | 否 |

`SampleScan` 不应作为试剂扫码位置。

## 3. 安全 IO 取证矩阵

> 每项必须经现场确认后才能填写真实极性 / 刷新 / 联锁；当前一律按「未知 / 断线默认不安全」处理。

- 所有安全 IO 的极性、刷新方式、断线默认策略、联锁动作。

- 主控 / SOCON / CAN / DCR55 / 制冷通讯断连判定与恢复规则。

### 8.3 必须进入 P0-05（坐标 / Z / 校准方案）的内容

- 坐标校准、复核、发布、冻结、回退、变更审计流程。

## 9. 现场取证方式

> 所有现场取证必须在可接触真实设备、厂商工程人员和现场资料的前提下，以**只读核对**为主，不自动连接、不自动扫描、不自动发送控制命令。

| 取证类别 | 取证方式 | 注意事项 |
|:----|:----|:----|
| 硬件标识 | 现场查看型号铭牌、序列号、适配器标识 | 序列号不入库 |
| 驱动 / 端口 | 现场查看设备管理器、驱动属性（不自动扫描脚本） | 真实端口号不入库 |
| CAN 参数 | 现场查看设备配置软件 / 出厂参数 | 真实 bitrate / channel 不入库 |
| NodeID / 映射 | 现场逐节点核对出厂配置 / 厂商工程人员确认 | 真实 NodeID / 泵映射不入库 |
| 串口参数 | 现场抓包或厂商资料复核 | 真实端口号不入库 |
| 协议行为 | 现场受控抓包（TX/RX）、记录原始帧（脱敏） | 原始帧不含敏感配置时才可记录 |
| 安全 IO | 现场逐项触发（急停 / 门 / 抽屉 / 限位 / 液位），记录来源、极性、上报、联锁 | 真实极性不入库，仅记录取证结论「已确认」 |
| 温控 / 制冷 | 现场实测温度、稳定时间、报警触发 | 阈值结论可入库，真实配置不入库 |
| 扫码 | 现场用测试码单次触发，记录返回格式、时序、EMPTY/INVALID | 测试码不得为真实患者 / 试剂码 |
| 坐标 / Z | 现场受控无液低速校准，按「回零→单轴→XY→双针→目标点」顺序 | 真实坐标不入库，仅记录校准状态 |

> 现场敏感参数（真实端口 / IP / NodeID / CAN 参数 / 坐标 / 泵映射 / 极性）仅保存于受控本机工程配置，**不进入 Git**。

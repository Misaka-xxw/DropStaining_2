## 1. 文档定位与边界

### 1.1 目的

本文为 P0-05 的设计草案，用于定义：

- 真实运动坐标需要采集和受控管理的参数结构；
- X / Y / Z / Z1 / Z2 轴映射、单位、方向和换算的现场确认方法；
- 双针几何、目标点、回零、单轴和无液低速空跑的分阶段校准方案；
- 校准记录、评审、发布、冻结、回退和审计规则；
- P1-08 前必须关闭的门禁和现场待确认项。

### 1.2 本阶段允许事项

- 只读分析 `DigitalTwinXY-v1` CSV 基线；
- 复用已提交的现场取证清单与安全门禁草案；
- 定义参数模板、现场 SOP、验收记录和阻塞关系；
- 将所有未知项明确标注为「待现场确认」。

### 1.3 本阶段禁止事项

本阶段不得：

- 连接、枚举或扫描 USB、COM、CAN、TCP 或任何真实设备；
- 打开串口、CAN 或网络连接；
- 加载或反射 SOCON SDK，实例化 SDK 类型或运行厂商 Demo；
- 调用 `Connect`、`OpenPort`、`Init`、`Move`、`LiqDet`、`Aspirate`、`Dispense`、`WaitActionDone`；
- 执行回零、点动、单轴运动、双针运动、目标点运动或湿实验；
- 把 SDK 默认值、历史样例或数字孪生值写成现场真实值；
- 在 Git 文档中记录真实坐标、真实 Z、真实软限位、真实 NodeID、真实 SetPerMM、真实安全位或其他敏感现场参数。

### 1.4 事实等级

| 等级 | 定义 | 本文用法 |
|:----|:----|:----|
| 已确认（仓库基线） | 已进入当前仓库且有来源可追溯 | 仅指 `DigitalTwinXY-v1` 的数字孪生 XY 数据与既有架构门禁 |

## 2. 当前已知 XY 基线说明

### 2.1 基线来源与状态

当前唯一数字孪生 XY 基线为：

```text
coordinate-baseline/position_coordinates_with_slide_rectangles_numbered.csv
```

对应坐标版本：

```text
DigitalTwinXY-v1
```

已知状态：

- CSV 共 90 行；项目上下文记录为 ExecutableTarget 70、ReferenceOnly 20、Rejected 0；
- 记录包含 `x_mm`、`y_mm`、类别、名称、行列和几何外形信息；
- 覆盖试剂区、玻片通道、清洗孔、洗针头、混合液体配液区、混匀电机、废液孔、排毒孔、A/B 液、相机和机械臂相对位置等数字孪生对象；
- 当前坐标状态仍为 `Draft / MockOnly / Unverified`；
- CSV 中的 X/Y 是设计与模拟基线，**不是现场测量值，也不是已校准真实坐标**。

### 2.2 基线使用原则

1. `DigitalTwinXY-v1` 仅作为目标命名、相对布局和现场核对的起点。
2. 现场不得直接把 CSV 的 X/Y 下发给真实设备。
3. 每个目标点必须建立「基线记录 → 现场测量记录 → 复核结果 → 受控版本」追溯链。
4. 若现场布局与基线不一致，不得临时镜像、旋转、平移或增加人为偏移；必须形成偏差记录并走受控变更评审。
5. 任何现场真实值仅进入受控本机工程配置或批准的校准存储，不进入 Git 文档。

### 2.3 数字孪生中的双针事实

现有资料与项目上下文仅给出数字孪生名义关系：

```text
Needle1 / Z1 = (0, 0)
Needle2 / Z2 = (0, +25 mm)
```

该 25 mm 仅为数字孪生名义偏移。真实针间距、偏移方向、装配误差、动态挠度和碰撞包络均为**待现场确认**。

## 3. 坐标单位与换算规则

### 3.1 单位边界

- `DigitalTwinXY-v1` 的 X/Y 字段以 mm 表达；这是数字孪生数据语义，不代表控制器当前已按 mm 工作。
- 厂商资料说明：轴在正确设置 SetPerMM 后可按 mm 工作，否则可能使用 step。
- X、Y、Z、Z1、Z2 各轴的真实 SetPerMM、控制器原始单位、回读单位和精度均为**待现场确认**。
- 未确认单位与换算前，禁止将任何数字孪生坐标转换为真实运动命令。

### 3.2 待评审的换算模型

未来现场确认后，每个轴应使用受控参数进行换算：

```text
轴物理位置 = 轴原始计数 ÷ 该轴已确认的每毫米计数
轴命令计数 = 目标物理位置 × 该轴已确认的每毫米计数
```

其中以下内容全部为**待现场确认**：

- 各轴每毫米计数 / SetPerMM；
- 正负方向；
- 零点定义与坐标原点；
- 原始计数与物理位置的偏置；
- 命令值和回读值的精度、舍入与容差；
- 是否存在机械传动间隙、回差补偿或方向相关补偿；
- 控制器断电、回零或故障后的坐标有效性。

### 3.3 坐标变换约束

`DigitalTwinXY-v1` 当前假设数字孪生 X/Y 与机械 X/Y 一致，不允许应用层静默增加镜像、旋转、缩放或平移。若现场证明该假设不成立，应：

1. 标记当前基线不适用；
2. 保持真实动作 fail-closed；
3. 记录偏差与证据；
4. 经工程、安全和软件评审后建立新的受控坐标版本；
5. 禁止现场人员用临时偏移绕过版本管理。

## 6. 双针几何与 25 mm 偏移验证方案

### 6.1 验证目标

验证数字孪生名义关系 `Needle2 = Needle1 + (0, +25 mm)` 是否与现场实际一致，并建立：

- 针尖静态间距：待现场确认；
- 偏移方向与符号：待现场确认；
- X 方向偏差：待现场确认；
- 两针 Z 零点与针尖高度差：待现场确认；
- 装配误差、回差和重复定位误差：待现场确认；
- 单针和双针运动碰撞包络：待现场确认；
- 双针同时进入相邻目标的可行性：待现场确认。

### 6.2 未来现场验证步骤（本阶段不执行）

1. **静态机械核对**：设备断能或处于厂商批准的安全状态，使用经校验量具测量两针名义间距；量具、方法和容差均待现场确认。
2. **轴映射核对**：确认 Z1 / Z2 与 Needle1 / Needle2 一一对应；真实映射待现场确认。
3. **只读位置核对**：在不运动条件下读取各轴位置并与机械状态比对；读取接口和判定规则待现场确认。
4. **单针低风险验证**：仅在 P1-08 前置全部满足且另行授权后，分别验证两针的方向、回读和重复性；速度、加速度和行程待现场确认。
5. **共享 XY 验证**：确认共享平台运动时两针相对几何保持稳定；允许偏差待现场确认。
6. **边界目标验证**：在无液、无样本、无试剂、无玻片条件下验证最不利目标的碰撞包络；测试点和容差待现场确认。
7. **结果评审**：任何偏差不得以临时补偿直接放行；应更新受控几何模型并重新评审。

### 6.3 通过条件

以下条件全部满足前，双针几何状态保持 Unverified：

- 实际间距、方向和 Z 高度差完成现场测量：待现场确认；
- 静态与重复定位容差已批准：待现场确认；
- 单针和双针碰撞包络已批准：待现场确认；
- 两针目标适用性和禁止组合已形成矩阵：待现场确认；
- 校准结果已评审、发布并冻结：待现场确认。

- 洗针液路和排废联锁未验证前禁止启用真实洗针动作。

### 7.4 DAB 位

- A/B 液、配液区与实际 DAB 工位的对应关系：待现场确认；

- DAB 目标未独立评审前不得由相邻数字孪生点推算。

### 7.5 混合区

- 核对混合液体配液区、混匀电机与真实物理位置：待现场确认；
- 核对针具、容器、混匀机构在静止和运行状态的碰撞包络：待现场确认；
- 混匀机构未锁定为安全状态前禁止机械臂进入；
- 进入高度、加液高度、退出顺序、速度和加速度均待现场确认；
- 运动与混匀时序未验证前禁止并发。

### 8.2 未来验证顺序

1. 仅做配置、轴映射、状态和安全 IO 的只读复核。
2. 核对机械位置与控制器位置有效性；判定规则待现场确认。
3. 按厂商批准顺序逐轴回零；轴顺序待现场确认。
4. 每次只允许一个轴进入验证状态，其他轴保持安全状态。
5. 使用经批准的最低风险速度和加速度验证短距离正/反方向；数值待现场确认。
6. 比对命令方向、物理方向和位置回读；容差待现场确认。
7. 验证停止、超时、通讯中断和安全 IO 触发后的 fail-closed 行为；触发方式须经安全评审。

- 受控配置版本与现场记录不一致；
- 监护人、停止手段或审计记录不可用。

终止后不得自动重试、自动续跑或自动解除门禁。

## 9. 无液低速空跑校准流程

> 本节定义未来流程，不授权当前执行。

### 9.1 阶段顺序

```text
只读状态复核
→ 回零验证
→ 单轴短距离验证
→ XY 空域验证
→ Z1 / Z2 单针安全高度验证
→ 双针静态几何与共享 XY 验证
→ 单类目标点验证
→ 最不利目标与退出路径验证
→ 记录评审
```

任何阶段失败均退回安全保持，不得跳级。

### 9.2 空跑条件

- 无样本、无真实试剂、无玻片、无液体；
- 针路排空状态和处置规则待现场确认；
- 速度、加速度、单次最大位移和观察停顿均待现场确认；
- 仅使用已批准坐标版本和本次校准快照；
- 每个目标先验证 Z 安全高度，再验证 XY，再验证接近/离开；
- 不执行液面检测、吸液、加液、洗针泵、混匀或排废动作；
- 不验证生产节拍，不以空跑通过代表湿实验可用。

### 9.3 目标点验证策略

1. 每类目标先选一个经评审的基准点；具体点位待现场确认。
2. 基准点通过后验证该类最不利边界点；判定方法待现场确认。
3. 针对 Needle1、Needle2 分别记录可达性和碰撞包络。
4. 双针同时可达场景必须单独评审，不从单针结果推定。
5. 任何临时偏移必须视为偏差，不得保存为正式校准值。

## 11. 校准记录、发布、冻结、回退与审计规则

### 11.1 校准记录最小字段

| 字段 | 要求 |

| 参数类别 | 轴映射 / 单位 / Z / 速度 / 加速度 / 限位 / 几何 / 目标点 |
| 真实值存储位置 | 仅受控本机工程配置；位置和访问策略待现场确认 |
| 校准状态 | Unverified / Measured / Reviewed / Approved / Rejected / Superseded；枚举待评审 |
| 执行人 / 复核人 / 批准人 | 角色分离；身份与签署方式待现场确认 |

| 证据引用 | 脱敏记录、照片或报告的受控引用，不含真实敏感参数 |
| 回退版本 | 明确可回退的上一个 Approved 版本 |

### 11.2 状态流转草案

```text
Draft / Unverified
→ Measured
→ Reviewed
→ Approved
→ Enabled + Frozen Snapshot
```

任一阶段可进入 `Rejected`；被新版本替代后进入 `Superseded`。状态名称和权限模型为**待评审**，在实现前不得视为现有正式状态机。

### 11.3 发布与冻结

- 真实参数与 DigitalTwin 基线必须分版本管理；
- 未经双人复核和批准不得发布；
- 已发布版本不可原地改写；任何调整必须创建新版本；
- 每次运行或空跑绑定不可变坐标快照；
- 校准状态不是单个布尔值，应能追溯到轴、目标点、针和版本；
- 任一依赖参数改变时，相关目标点必须降级为 Unverified 或待复核；依赖关系待评审。

### 11.4 回退

- 回退必须显式选择已批准版本并记录原因、操作者、批准人和时间；
- 回退后必须重新执行启动前只读复核；
- 不得因回退而绕过新出现的安全门禁或现场变更；
- 若机械结构、针具、驱动、轴映射或限位发生变化，旧版本不得直接启用；
- 无可信回退版本时保持 fail-closed。

### 11.5 审计

必须审计：

- 草案、测量、复核、批准、启用、停用、冻结和回退；
- 参数类别和版本差异，不在 Git 审计中暴露真实敏感值；
- 每次校准/空跑的门禁求值、异常、停止与人工确认；
- 任何临时偏移或现场偏差请求；
- 未知结果、通讯中断和恢复处置。
